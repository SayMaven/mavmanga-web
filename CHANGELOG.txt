================================================================================
  RINGKASAN TEKNIS
================================================================================

PRINSIP OPTIMASI YANG DITERAPKAN:
  1. GPU Compositing  : transform: translateZ(0) / will-change: transform
                        Memindahkan elemen ke layer GPU terpisah sehingga
                        animasi tidak memblokir main thread (critical di HP
                        dengan GPU terbatas)

  2. Image Loading    : loading="lazy" untuk gambar di bawah fold,
                        loading="eager" + fetchPriority="high" untuk gambar
                        LCP (Largest Contentful Paint) agar skor Core Web
                        Vitals meningkat

  3. React Patterns   : React.memo mencegah re-render komponen jika props
                        tidak berubah; useMemo cache hasil kalkulasi berat;
                        useCallback cache fungsi agar referensi stabil

  4. Event Optimization: passive: true pada scroll/wheel listeners agar
                        browser bisa scroll tanpa menunggu JavaScript selesai

  5. RAF Throttling   : requestAnimationFrame pada scroll handler reader
                        memastikan kalkulasi DOM hanya terjadi sekali per
                        frame (60fps) bukan tiap event scroll

  6. Bundle Size      : Penghapusan react-markdown + remark-gfm menghemat
                        ~60KB pada bundle halaman manga detail

  7. API Efficiency   : Menghapus duplikasi fetch getMangaRecommendations
                        yang sebelumnya dipanggil dua kali untuk data yang sama

  8. Stacking Context : Perbaikan bug dropdown tertutup elemen lain dengan
                        menghapus "contain: layout style" yang secara implisit
                        membuat stacking context baru

  9. Memory Safety    : Cancelled flag pada async fetch di ReaderViewer mencegah
                        setState setelah komponen unmount (memory leak)
